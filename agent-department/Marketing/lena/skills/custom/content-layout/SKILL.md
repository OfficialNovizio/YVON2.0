## Purpose

Copy without correct platform structure fails before it's read. A wall of text on Instagram isn't ignored because the words are bad — it's ignored because the structure is wrong. This skill defines the layout contract for each platform.

---

## Instagram

### Structure
```
[Hook — line 1 only, no period, creates tension or curiosity]

[Body — 2–3 short paragraphs, one idea each]

[CTA — single line, direct]

[Tags — separate line, 3–5 max]
```

### Rules
- Line 1 must work as a standalone sentence — it's the only thing visible in the feed before "more"
- Always line-break between paragraphs — never a wall of text
- Emoji: Novizio — none or max 1, end of line only. Hourbour — none.
- Never end a line with a period if the next line continues the thought
- Tags go after a line break, not inline with body text

### Length
| Venture | Target Length | Reason |
|---------|--------------|--------|
| Novizio | 100–180 words | Editorial — doesn't over-explain |
| Hourbour | 80–140 words | Professional — gets to the point |

## LinkedIn

### Structure
```
[Hook — first 1–2 lines, visible before "see more", creates a knowledge gap]

[White space]

[Body — 3–5 short paragraphs, max 3 sentences each]

[White space]

[CTA — single line]
```

### Rules
- Hook must stand alone: a cold reader who sees only line 1 should want to click "see more"
- Never use "I" as the first word of the hook — algorithm and reader both prefer a concrete claim
- No exclamation marks — on either venture
- White space is structural, not decoration — short paragraphs, visible breathing room
- Hourbour primary platform. Professional tone throughout. No casual register.
- Novizio on LinkedIn: thought-leadership framing only (industry perspective, not product push)

### Length
| Venture | Target Length | Reason |
|---------|--------------|--------|
| Novizio | 200–300 words | Thought-leadership register — needs development |
| Hourbour | 250–400 words | Professional depth — evidence-driven |

## Email

### Anatomy
```
Subject line: [40–50 chars, no clickbait, matches content]
Preview text: [Complements subject — does not repeat it]

[Greeting — 1 line, name or "Hey [name]"]
[Lead — 1–2 sentences, the hook and why this matters now]

[Body — 3–4 short paragraphs]

[CTA — single button or link, clear action]

[Sign-off — name + one-line brand signature]
```

### Rules
- Subject line cannot exceed 50 characters — mobile truncates beyond that
- Preview text must add information, not restate the subject line
- Body paragraphs max 4 sentences — readers scan, not read
- Single CTA only — multiple links split attention and reduce clicks
- Novizio email: treat as editorial correspondence, not marketing blast — never "SALE" or "LAST CHANCE" in subject
- Hourbour email: direct, outcome-focused — subject line states the benefit, not the feature

### Subject Line Patterns
| Venture | Pattern | Example |
|---------|---------|---------|
| Novizio | [Quiet revelation or identity statement] | "This season's edit is different." |
| Hourbour | [Specific outcome for the reader] | "Your hours, tracked to the minute." |

## TikTok / Reels Script

### Structure
```
[Hook — 0–3 seconds: spoken or text overlay, creates immediate tension]

[Build — 3–20 seconds: evidence, story, context]

[Resolution — 20–30 seconds: payoff + CTA]
```

### Rules
- Hook must create tension in under 3 seconds — the viewer decides to stay in the first cut
- Text overlay must be max 6 words per screen — reading and watching compete
- Lena writes: spoken script + text overlay copy
- Atlas handles: visual direction. Lena does not specify shot types.
- CTA spoken, not just text — "Follow for..." or "Link in bio for..." as last line of script
- Novizio: visual hook relies on Atlas's brief. Lena's script supports, not leads.
- Hourbour: problem-solution-proof arc. Hook names the pain, build proves the fix, resolution delivers the CTA.

### Script Format
```
[0-3s] HOOK: "[spoken line]" / OVERLAY: "[max 6 words]"
[3-20s] BUILD: "[2–3 spoken sentences]"
[20-30s] RESOLUTION: "[1 sentence payoff] + [CTA spoken]"
```

## Cross-Platform Rules (All)

1. Never write copy that only works as text — think about how it looks in the layout
2. One idea per paragraph. Multiple ideas in one block = reader exits.
3. First line of every platform carries the entire weight — if it doesn't earn the next line, the rest is wasted
4. Match length to platform, not to how much there is to say
5. Formatting is part of the copy. A perfectly written wall of text is a failure.
